## ToolbarControl

- This mod vendors (compiles directly into its own plugin) the ToolbarControl
  client library by linuxgurugamer, from
  https://github.com/linuxgurugamer/ToolbarControl. This gives players who
  also install the optional "ToolbarController" companion mod (and/or Blizzy's
  Toolbar Continued) a freely repositionable icon; without either installed,
  the mod falls back to the stock AppLauncher exactly as before, unchanged.
- Vendored at `src/CustomScienceContracts/Vendor/ToolbarControl/`:
  `BlizzyOptions.cs`, `ConfigInfo.cs`, `InstallChecker.cs`, `IntroWindow.cs`,
  `Log.cs`, `RegisterToolbar.cs`, `RegisterUsage.cs`, `Settings.cs`,
  `ToolbarControl.cs`, `ToolbarWrapper.cs`.
- ToolbarControl is licensed under the GNU Lesser General Public License
  version 3.0. The LGPL v3.0 license text is included at
  `LICENSES/LGPL-3.0.txt`.
- Two portions carry their own, different licenses, both retained/reproduced
  as shipped upstream:
  - `ToolbarWrapper.cs` is Copyright (c) 2013-2016 Maik Schreiber, under a
    2-clause BSD license. Text at
    `LICENSES/BSD-2-Clause-ToolbarWrapper.txt`.
  - `InstallChecker.cs` is based on the InstallChecker from the Kethane mod
    (Copyright Majiir), released CC0 1.0 Universal / public domain. Text at
    `LICENSES/CC0-1.0.txt`.
- Modifications made for CustomScienceContracts (all marked inline with a
  `[CSC]` comment at the change site, so a future upstream diff stays easy):
  - `ToolbarControl.cs`: removed the optional DDS icon-loading code path (and
    the `using DDSHeaders;`/`LoadTextureDXT` it required) since this mod only
    ships PNG icons.
  - `BlizzyOptions.cs`, `IntroWindow.cs`: replaced the `ClickThruBlocker`
    click-through-guard calls (from the separate "ClickThroughFix" library,
    not vendored) with plain `GUILayout.Window`, since the only two call
    sites are ToolbarControl's own rarely-shown internal settings/intro
    popups, not any CustomScienceContracts window.
  - `RegisterToolbar.cs`: removed an unused `using ClickThroughFix;` line
    (the file never called into it).
  - Not vendored: `AssemblyVersion.cs` and `Properties/AssemblyInfo.cs`
    (would duplicate assembly-level attributes already generated by this
    project's own `.csproj`) and `Resources/` (icon assets for
    ToolbarControl's own settings UI; not referenced by any of the vendored
    code paths this mod actually uses).
